Button Tiers

Two button sizes driven by natcaDefaults. Toolbar buttons are compact/small by default. Action buttons in <v-card-actions> and dialogs automatically get default sizing (NAT-281 fix).

Button Colors

VariantVuetify UsageWhen to Use
color="primary"Main action (save, submit, confirm)
variant="outlined"Secondary actions (export, edit)
variant="outlined" color="error"Destructive (delete, remove). Outlined to distinguish from primary.
variant="text"Cancel, dismiss, tertiary actions
variant="text" color="primary"Inline navigation (view details, see more)
Dark mode note: Primary is red (#CE0E2D) in dark mode. Danger buttons use variant="outlined" color="error" to be visually distinct from filled primary buttons. On hover, danger fills solid red for emphasis.
Toolbar Tier (default VBtn)

density: compact, size: small. Filters, table actions, toolbars.

Action Tier (VCardActions / Dialogs)

density: default, size: default. Auto-applied inside v-card-actions.

How it works: natcaDefaults.VBtn sets size: 'small', density: 'compact' globally. natcaDefaults.VCardActions.VBtn overrides to size: 'default', density: 'default'. No manual sizing needed.

Forms

All form fields use outlined variant, compact density, primary focus color via natcaDefaults.

Create Email Account
natcaDefaults handles:
VTextField → outlined, compact, primary focus
VSelect → outlined, compact, primary focus
VAutocomplete → outlined, compact, primary focus
VCheckbox → compact, primary, hideDetails auto
VSwitch → compact, primary, inset, hideDetails auto
VTextarea → outlined, compact, primary focus
Never do this:
<v-text-field variant="outlined" density="compact" color="primary" />
All three props are already set by natcaDefaults. Just use:
<v-text-field label="Email" />
