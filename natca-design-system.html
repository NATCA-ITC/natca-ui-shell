Toasts & Notifications

Universal pattern for all apps. Toasts stack bottom-right, auto-dismiss after 5 seconds (errors persist until dismissed). All apps share the same toast component from @natca/ui-shell.

Usage Guidelines

Do: Use toasts for transient feedback — save confirmations, upload completions, background sync results. Keep messages under two lines.
Don't: Use toasts for critical actions that need confirmation (deleting data, filing grievances). Use a modal dialog instead.
When to use each type:
Success — Action completed (saved, uploaded, synced)
Error — Action failed (persists until dismissed)
Warning — Attention needed (deadline approaching, data issue)
Info — System notice (maintenance, updates, announcements)
