Authenticated Member & Admin Views

Three shell designs split by audience: Auth A is the admin portal (facility reps, national staff, officers — anyone with elevated permissions), Auth B is the member/BUE shell, Auth C is a hybrid merging the best of both. Auth A Full Admin is the future unified admin portal; Auth A Light and Auth B serve regular members. Auth C combines B's navy top bar and tab nav with A's optional sidebar. All use data-theme="dark". The sidebar toggle on Options A and C is live — click ☰ to collapse.

Auth Option A Command Shell — Admin portal shell for facility reps, national staff, officers, and anyone with elevated permissions. Deep navy top bar, icon-rail collapsible sidebar. Dense, data-forward.

Auth Option A — Full Admin: Deep navy top bar with sidebar that collapses to icon-rail. ZJX facility badge provides location context. Full Admin is the future unified admin portal shell — all admin tools (Hub, MyNATCA admin, GATS management, etc.) converge here. The Light variant strips the sidebar for single-purpose or member-facing apps. Best for: Admins with elevated permissions — facility reps, national staff, officers. This is the Option 2 → Option 3 path: start with Full Admin as the primary admin shell, then evolve it into the unified portal.

Auth Option B — Horizontal Nav Shell: Member-facing shell for all BUEs. App switcher lives in the system name chip ([⊞ AppName ▾]) — click to switch between MyNATCA apps. All navigation in a full-width tab bar. The contextual second row has two modes: Sub-links mode shows top-level sections within the app (used at section roots, e.g., Overview / Facility Stats / Recent Activity). Breadcrumb mode shows a path trail when deep in a record (e.g., GATS / ZJX Grievances / Case #2026-041). The app controls which mode renders based on route depth. Maximizes horizontal content width — ideal for data tables, GATS case files, and bidding grids.
Auth Option C Hybrid Shell — Best of A and B. Navy top bar with compact logo + app switcher chip. Tab nav row with icon+label tabs and "More ▾" overflow. Required breadcrumb/context row. Optional sidebar for admin pages. Sidebar collapses to icon-rail.

Auth Option C — Variant 1 (Hub Admin): Full hybrid shell with sidebar + tab nav. Navy top bar keeps the compact NATCA logo and app switcher chip ([⊞ Hub ▾]). The tab row provides primary navigation; the sidebar provides contextual sub-nav within the active section. Sidebar collapses to a 52px icon-rail via the toggle button. Breadcrumb row is always present with a right-side slot for contextual metadata (member counts, sync times, deadlines). Best for: Hub admins, facility reps, and national staff who need both top-level section switching (tabs) and deep sub-navigation (sidebar).

Auth Option C — Mobile: Navy top bar with NATCA logo, divider, and wordmark — matching the desktop topbar styling. Breadcrumb row below the top bar includes a hamburger button for drawer toggle and location context. Bottom tab bar is pinned to the bottom of the viewport for quick section access (Dashboard, Bidding, Grievances, Docs, More). The drawer (hamburger tapped) shows the full sidebar nav sections grouped identically to desktop — no app switcher in the drawer since it's already accessible in the top bar. Sign Out anchored at the bottom. The drawer is a full-screen dark overlay.
